## 九、实现方式说明

### 内存追踪（In-Memory）

以下频控使用 **进程内存 dict** 记录时间戳，进程重启后清零：

- 点赞频控 (`like_service.py`)
- 评论频控 (`comment_service.py`)
- 投票创建/投票频控 (`vote/service.py`)
- 图床上传频控 (`image_hosting/service.py`)
- 照片墙贴图/操作频控 (`photowall/service.py`)

**特点**：轻量、零数据库开销，但不适合多进程部署（每个 worker 独立计数）。

### 数据库精确统计

以下限额通过 **数据库查询** 精确统计：

- 博客每日发文数 (`Blog.created_at >= start_of_day`)
- 申诉每日次数 (`AdminActionAppeal.created_at >= start_of_day`)
- 剪贴板总数 (`ClipBoard.author_id.count()`)
- 照片墙照片总数 (`PhotoWallItem.author_id.count()`)
- 投票总数 (`VoteRecord.user_id.count()`)
- 签到 (`DailyCheckIn` 唯一约束)

**特点**：精确、跨进程一致，但有查询开销。

### 唯一约束

以下限制通过数据库 **UNIQUE 约束** 实现：

- 同一用户对同一博客只能有一条点赞记录 (`BlogLike`)
- 同一用户对同一投票只能投一次 (`VoteRecord`)
- 同一用户同一天只能签到一次 (`DailyCheckIn`)
