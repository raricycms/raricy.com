# 内容引用语法 `[@ ]` 使用指南

在博客文章中使用 `[@<内容ID>]` 语法，可以在文章渲染时自动嵌入剪贴板文字、投票组件和图床图片。系统根据 ID 长度自动识别内容类型。

---

## 前置条件

- 需要已登录且通过邀请码认证（Core 用户及以上角色）
- 嵌入的内容（剪贴板/投票/图片）需为公开状态，或你本人是内容作者
- 单篇博客最多嵌入 **50 个**内容引用

---

## 一、三种引用类型

| 语法 | ID 长度 | 内容类型 | 渲染效果 |
|------|---------|----------|----------|
| `[@12345678]` | 8 位 | 云剪贴板 | 将 ID 对应剪贴板的正文直接替换到文章中 |
| `[@123456789]` | 9 位 | 投票 | 渲染为可交互的投票小部件 |
| `[@1234567890]` | 10 位 | 图床图片 | 渲染为 Markdown 图片 `![](/image/i/ID)` |

ID 长度不在 8-10 之间的 `[@...]` 不会被处理，以原文形式保留。

---

## 二、剪贴板嵌入（8 位 ID）

### 获取 ID

访问 `/clipboard/`，在列表中查看每个剪贴板的 8 位 ID，或从剪贴板详情页 URL `/clipboard/<ID>` 中获取。

### 使用方式

在博客 Markdown 正文中直接写入：

```
[@a1b2c3d4]
```

### 渲染行为

系统向后端请求剪贴板正文内容，将其**原样替换** `[@a1b2c3d4]` 标记。剪贴板中的 Markdown 格式会正常参与后续渲染（标题、列表、代码块等均生效）。

### 错误处理

如果剪贴板不存在、非公开或无权限访问，该位置会显示：`[剪贴板 a1b2c3d4 加载失败]`

---

## 三、投票嵌入（9 位 ID）

### 获取 ID

访问 `/vote/` 查看你的投票列表，或从投票详情页 URL `/vote/<ID>` 中获取。

### 使用方式

```
[@vOtE12345]
```

### 渲染行为

系统请求投票数据并在文章内渲染一个**可交互的投票小部件**，包含：

- 投票标题
- 选项列表
- **未投票用户**：可点击选项并提交投票
- **已投票用户**：显示各选项票数和百分比进度条，自己的选项高亮标注
- **已锁定投票**：显示锁定标记和结果，无法继续投票
- 底部"查看详情"链接跳转到完整投票页面

### 错误处理

如果投票数据加载失败，该位置会显示：`[投票 vOtE12345 加载失败，点击查看]`（可点击跳转到投票页）

---

## 四、图床图片嵌入（10 位 ID）

### 获取 ID

访问 `/image/` 查看你上传的图片列表，或在图片详情中获取 10 位 ID。

### 使用方式

```
[@AbCdEf1234]
```

### 渲染行为

系统将其替换为标准 Markdown 图片语法：

```markdown
![AbCdEf1234](/image/i/AbCdEf1234)
```

图片渲染后支持点击放大查看。

### 说明

图片嵌入不需要额外请求 API —— 系统直接拼接图片 URL，浏览器加载时自然请求 `/image/i/<ID>`。如果 ID 对应图片不存在或已被删除，会显示为损坏图片。

---

## 五、技术实现简述

博客详情页加载时的处理管道：

```
博客 Markdown 正文
        │
        ▼
ClipboardPreprocessor.preprocess()
  正则匹配 /\[@\s*(\w+)\s*\]/g
  按 ID 长度分类，并行获取数据
  将 [@id] 替换为对应内容
        │
        ▼
MarkdownRenderer.render()
  marked.parse() → HTML
  DOMPurify 清理 → 安全渲染
  highlight.js 代码高亮
  MathJax 数学公式渲染
        │
        ▼
VoteEmbed.init()
  扫描 .vote-embed 元素
  逐个渲染投票小部件
```

### 缓存

`ClipboardPreprocessor` 内部使用 `Map` 缓存已获取的内容，同一 ID 在文章中多次出现只请求一次。缓存上限 50 条。

### 并行请求

多个 `[@id]` 的剪贴板和投票数据**并行获取**，不会串行阻塞。图片无需请求。

---

## 六、注意事项

1. **非用户提及**：`[@ ]` 语法不是 @用户 功能，不能用来提及或通知其他用户。
2. **数量限制**：单篇文章最多替换 50 个引用标记，超出部分保留原文。
3. **权限检查**：私有剪贴板只有作者本人可以在文章中引用；其他人引用会加载失败。
4. **ID 大小写**：ID 区分大小写（包含大小写字母和数字），请完整复制。
5. **编辑器中不可预览**：在 Vditor 编辑器中编写时，`[@id]` 以文本形式显示，仅在发布后的博客详情页才会渲染为嵌入内容。
6. **剪贴板内容为纯文本替换**：嵌入的剪贴板 Markdown 会成为文章 Markdown 的一部分，注意避免引入破坏性的格式（如未闭合的代码块）。
