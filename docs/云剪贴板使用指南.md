# 云剪贴板使用指南

云剪贴板是 Raricy.com 提供的一款 Markdown 内容管理与复用工具。你可以在这里撰写文章、保存片段，并通过简单的语法将任意剪贴板内容直接嵌入博客正文中。

---

## 前置条件

使用云剪贴板需要已登录且通过邀请码认证（Core 用户及以上角色）。如果尚未认证，请先联系站长获取邀请码。

---

## 一、主页面

访问地址：`/clipboard/`

主页面展示你创建的所有剪贴板列表，每条显示标题和 8 位 ID。列表按创建时间倒序排列。

顶部提供两项操作：

- **搜索跳转**：输入剪贴板 ID，点击"前往"即可直接打开对应剪贴板详情页。
- **创建文章**：点击按钮进入新建剪贴板页面。

---

## 二、创建剪贴板

点击"创建文章"按钮进入 `上传云剪贴板` 页面。

| 字段   | 说明                          |
| ---- | --------------------------- |
| 标题   | 1–40 个字符，必填                 |
| 正文   | Markdown 格式，最多 50,000 字符，必填 |
| 是否公开 | 勾选则所有人可见，取消则仅作者可见           |

填写完成后点击"提交"按钮。创建成功后自动跳转到该剪贴板的详情页。

> 每个用户最多创建 **200 篇**剪贴板。

---

## 三、查看剪贴板

详情页会将 Markdown 正文渲染为富文本，支持以下特性：

- 标准 Markdown 语法（标题、列表、加粗、斜体、引用等）
- 代码块语法高亮（自动识别语言）
- 代码块一键复制按钮
- GFM 表格
- 任务列表（复选框）
- 图片点击放大
- 外部链接自动加 `rel="noopener noreferrer nofollow"` 并在新标签页打开

页面底部操作栏：

| 按钮       | 说明                    |
| -------- | --------------------- |
| 编辑文章     | 仅作者可见，点击进入编辑页         |
| 删除文章     | 仅作者可见，点击后确认删除       |
| 复制正文     | 将原始 Markdown 内容复制到剪贴板 |
| 返回云剪贴板主页 | 回到列表页                 |
| 返回上页     | 浏览器后退                 |

---

## 四、编辑剪贴板

编辑页面与创建页面共用同一表单，进入时自动填充已有内容。

修改标题、正文或公开状态后，点击"更新"按钮保存并返回详情页。

### Ctrl+S 快捷保存

编辑过程中可随时按下 **`Ctrl+S`**（macOS 为 `Cmd+S`）手动保存，无需离开编辑区。保存成功后页面右上角会弹出绿色提示"保存成功！"，你仍停留在编辑页面继续编辑。保存失败时会弹出红色错误提示。

### 自动保存

编辑页面提供"自动保存（每分钟）"开关，勾选后每隔 1 分钟自动保存一次，避免因意外关闭页面导致内容丢失。

- 默认关闭，需手动勾选开启。
- 勾选状态保存在浏览器本地，下次打开编辑页面时自动恢复。
- 自动保存成功/失败同样通过右上角 toast 提示，不打断编辑。
- 点击"更新"按钮提交后，自动保存会自动停止。

> 编辑页面顶部有提醒横幅，提示用户使用 Ctrl+S 手动保存或开启自动保存。

---

## 五、删除与恢复

删除操作为**软删除**——数据库将剪贴板标记为已删除，不会物理删除数据。如果你误删了剪贴板，可联系站长进行恢复。

---

## 六、公开与私密

- **公开**（默认）：任何人知道 ID 即可通过详情页查看内容，也能通过 `?raw` 接口获取原始数据。
- **私密**：仅作者本人可以查看。其他用户访问时返回 403。

私密剪贴板**不会被博客中的 `[@id]` 引用加载**（因为 `?raw` 接口会校验权限），因此不会泄露。

---

## 七、在博客中引用剪贴板

这是云剪贴板最强大的功能：你可以在博客文章中直接嵌入任意剪贴板的内容。

### 语法

```
[@剪贴板ID]
```

例如，假设你有一篇剪贴板 ID 为 `abc12345`，在博客正文中写入：

```markdown
今天我们来聊聊某个话题：

[@abc12345]

以上就是全部内容。
```

发布后，`[@abc12345]` 会被自动替换为该剪贴板的**完整 Markdown 正文**，并与博客正文合并渲染。

### 注意事项

- 每篇博客最多引用 **100 个**不同的剪贴板。超出部分将保持原样显示（即显示为 `[@id]` 原文）。
- 引用的剪贴板必须是**公开**的，私密切贴板无法加载。
- `[@id]` 的内部空格会被忽略，因此 `[@abc12345]` 和 `[@ abc12345 ]` 效果相同。
- 替换在浏览器端完成，不影响服务端存储的博客原文。

## 九、常见问题

### Q: 我可以修改别人的剪贴板吗？

不可以。只有作者本人可以编辑自己的剪贴板。

### Q: 删除后还能找回吗？

可以联系站长恢复。删除为软删除，数据仍保留在数据库中。

### Q: 为什么我在博客里引用的剪贴板没有显示？

可能的原因：

1. 该剪贴板为私密状态（仅作者可见，加载失败）
2. ID 拼写错误
3. 该博客引用了超过 100 个剪贴板，超出的部分不会加载

### Q: 剪贴板可以包含图片吗？

可以。支持标准 Markdown 图片语法 `![alt](url)`，详情页渲染后图片还支持点击放大查看。
