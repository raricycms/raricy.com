# 前端样式构建指南

本文档描述项目的 SCSS 目录结构、命名规范和 Sass + PostCSS 构建流程。

## 目录约定

- 源码目录：`app/static/scss/`
- 产物目录：`app/static/css/`
- 入口文件：`app/static/scss/main.scss`

## SCSS 目录结构

```
static/scss/
  abstracts/           # 设计令牌与工具（仅被引用，不产出 CSS）
    _variables.scss    # 颜色、间距、字号、圆角、阴影、z-index、断点
    _mixins.scss       # 常用 mixin（媒体查询、文本截断、渐变等）
    _functions.scss    # SCSS 函数（色彩调整、单位换算等）
    _breakpoints.scss  # 断点常量与响应式 mixin
  base/                # 全局基线
    _reset.scss        # Reset / Normalize
    _typography.scss   # 字体、字号体系、标题/段落/链接
    _forms.scss        # 表单控件基础样式
  layout/              # 布局与栅格
    _container.scss    # 容器宽度、边距
    _grid.scss         # 简易 Flex/Grid 栅格
    _header.scss       # 站点头部
    _footer.scss       # 站点底部
  components/          # 可复用组件（BEM 命名）
    _button.scss
    _navbar.scss
    _modal.scss
    _card.scss
    _alert.scss
    _form-controls.scss
  utilities/           # 高价值工具类（前缀 u-）
    _spacing.scss      # .u-mt-8 等
    _display.scss      # .u-hidden / .u-block 等
    _flex.scss         # .u-flex / .u-items-center 等
    _text.scss         # 文本截断、对齐、换行
  pages/               # 页面特有样式
    _home.scss
    _blog.scss
    _auth.scss
    _notifications.scss
    _story.scss
    _tool.scss
    _sitemap.scss
  themes/              # 主题（可选）
    _dark.scss
  vendors/             # 第三方样式覆写
    _pygments.scss
  main.scss            # 入口文件：按顺序引入各层
```

## main.scss 引入顺序

固定顺序确保特异性与覆盖关系稳定：

1. `abstracts` — 变量/函数/mixin/断点
2. `base` — 全局基线
3. `layout` — 布局与栅格
4. `components` — 通用组件
5. `utilities` — 工具类
6. `pages` — 页面特有样式
7. `themes` — 可选主题
8. `vendors` — 第三方覆写

## 命名规范

- **组件（BEM）：** `block`、`block__element`、`block--modifier`
  - 示例：`card`、`card__title`、`card--featured`
- **工具类（Utility）：** 前缀 `u-`
  - 示例：`.u-mt-8`、`.u-flex`、`.u-items-center`、`.u-hidden`
- SCSS 嵌套：最多两层，避免高特异性选择器
- 禁止 `!important`（utilities 层除外）
- 变量优先：颜色、间距、字号全部走设计令牌

## 工具链

### 依赖

```bash
npm install sass postcss autoprefixer cssnano postcss-cli
```

### 浏览器目标（Browserslist）

```
> 0.5%
last 2 versions
not dead
```

### 开发模式

编译 SCSS + 生成 SourceMap + 监听文件变化：

```bash
sass --style=expanded --source-map static/scss/main.scss static/css/main.css --load-path=static/scss --watch
```

### 生产构建

编译 + 自动加前缀 + 压缩：

```bash
# 1) Sass 编译（无 SourceMap）
sass --style=expanded static/scss/main.scss static/css/main.css --no-source-map --load-path=static/scss

# 2) PostCSS：autoprefixer + cssnano
postcss static/css/main.css --use autoprefixer --no-map | postcss --use cssnano --no-map > static/css/main.min.css
```

### 模板引用策略

- 开发：模板引用 `static/css/main.css`
- 生产：模板引用 `static/css/main.min.css`
- 第三方样式（如 Pygments）独立 `<link>`，需要覆写时在 `vendors/` 中进行

## 组件与页面边界

- 页面专属样式只能写在 `pages/`
- 复用性出现时，将样式上提到 `components/` 或 `layout/`
- 组件内不依赖页面上下文（避免耦合）

## 第三方样式策略

- 不改动第三方原文件，必要时在 `vendors/` 增量覆写
- 如完全移除 Bootstrap：用 `layout/` 与 `components/` 提供的替代类
- 如保留 Bootstrap：置于 `static/vendor/bootstrap/`，在其后加载 `main.css` 逐步替换

## 构建脚本

项目提供了便捷脚本：

```bash
# 开发（watch 模式）
./scripts/build_scss.sh dev

# 生产构建
./scripts/build_scss.sh prod
```
