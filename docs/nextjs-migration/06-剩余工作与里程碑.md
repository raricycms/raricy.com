## 权限对齐说明（已按 Flask 逐条核定）
- `/blog/upload`、`/blog/[id]/edit`：`@authenticated_required`（core+）→ 未登录/非核心 403（实测 403）。
- `/fish`、`/fish/transactions`：`@login_required`（任意登录用户）→ 未登录 307 跳登录（实测 307）。
- `/api/admin/notify-user`：`@owner_required` → 仅站长（isOwner）。
- `/api/users/[id]/ban-history`：`@authenticated_required` → core+（isCoreUser）。
- `/image/admin`、`/api/images/admin/[id]`：`@owner_required` → 仅站长（实测 403）。
- 博客编辑：Flask 为**仅作者本人**（admin 也不可），已忠实复刻。

## 🔴 上线前仍需满足的运行时前置条件（部署项，非代码缺口）

1. **时间戳格式**：真实库需先跑 `scripts/normalize-datetimes.mjs` 转 ISO（prod.db 已转）。
   硬切换后仅 Next 写库，写入即 ISO，长期一致。见 [03](03-数据库映射与陷阱.md)。
2. **经济系统生产密钥**：`.env.production` 必须填入 Flask 生产环境实际加密
   `fish_api_key_encrypted` 用的密钥（本仓库 `.env` 的 SECRET_KEY 解不开真实数据）。
   否则 465 个用户鱼干账户解不开、经济系统 fail-closed(503)。**这是 Flask 同样具备的依赖。**
3. **账户微服务可达且已联调**：TS 客户端 + Fernet 兼容已验，但尚未对运行中的真实
   account-service 端到端联调（本环境无法起该服务）。切换前需真连跑通投喂/发鱼一次。
4. **data/ 目录随部署提供**：头像（data/avatars）、图片（data/images）由 Next 原生读取，
   通过 `.env` 的 `AVATARS_DIR` / `IMAGE_UPLOAD_FOLDER` 指向。部署时挂载真实数据卷。
5. **会话不互通**：切换后用户需重新登录一次（密码不用改，session_version 从 0 起）。
6. **内存限频**：单进程有效；多实例部署需换 Redis。
7. **SQLite 并发写**：高并发写是 SQLite 固有短板，长期建议迁 Postgres（[03 §7](03-数据库映射与陷阱.md)）。

## 删除 Flask 的操作顺序（建议）
1. 部署 web-next 到生产，满足上方 1–4 前置条件，与 Flask 并行灰度。
2. 冒烟 + 灰度验证所有模块（尤其经济系统真连账户服务）。
3. 切流量到 Next；观察一段时间。
4. 确认无回退需求后，删除 `app/`、`run.py`、`account-service/` 的 Flask 侧代码
   （保留 `instance/`→`data/` 数据、`docs/`、`web-next/`）。
