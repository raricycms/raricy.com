## 一次迁一个模块的检查清单（复制去用）

```text
[ ] service：纯函数，读写走 prisma，限频走 rate-limit，软删除/唯一约束照搬
[ ] route：判权（getCurrentUser + 角色 helper），返回 { code, message, ...data }
[ ] page：Server Component 直接 await service；交互部分抽 'use client'
[ ] 边界：401/403/404/429 与 Flask 行为对齐（但 API 一律 JSON，不返回 HTML 403）
[ ] 数据：涉及时间写入确认走 ISO；JSON 字段 parse/stringify
[ ] 冒烟：curl 正常/异常路径各一遍
```

## 上线加固进展（2026-07-15 已做 + 已实测）

- ✅ **SQLite 并发**：全库开 **WAL**（读写不互阻）；`DATABASE_URL` 加 `connection_limit=1&socket_timeout=30`
  序列化写、并发排队。实测 **60 并发读写 → 60×200、0 错误**（加固前有 14 个 Socket-timeout 500）。
- ✅ **CSRF**：新增 `src/middleware.ts` 对 `/api/*` 写请求做 Origin 同源校验（跨源写 → 403，已实测），补齐 Flask 原先没有的 CSRF 层。
- ✅ **经济系统 Fernet 解密**：TS 版与 Flask `AES.py` 派生方式一致，合成密钥往返 **MATCH**（字节级兼容）。
- ✅ **生产构建**：`output:'standalone'` + `serverExternalPackages`（sharp/fernet/prisma），产出 `.next/standalone/server.js` 可直接 PM2/systemd 起。
- ✅ **生产配置模板**：`.env.production.example`，含下方 🔴 密钥前置条件的醒目警告。

## 🔴 上线前置条件（实测发现，务必满足）

- **经济系统密钥必须照搬 Flask 生产值**：实测本仓库 `.env` 的 `SECRET_KEY` 解不开真实库里
  **任何一个** `fish_api_key_encrypted`（0/8），说明真实数据是用【另一个】密钥加密的（生产的
  `FISH_ENCRYPTION_KEY` 或历史 `SECRET_KEY`）。部署 Next 时**必须**在 `.env.production` 填入
  Flask 生产环境实际用于加密这批数据的那个密钥，否则 465 个用户的小鱼干账户全部解不开、
  经济系统整体 fail-closed(503)。**这是 Flask 同样具备的依赖，不是 Next 引入的问题。**
- **账户微服务必须可达且已联调**：TS 客户端结构完成、Fernet 兼容已验，但**尚未对运行中的
  真实 account-service 端到端联调**（本会话无法起该服务）。上线前需真连一次跑通投喂/发鱼。

## 已知风险登记

1. **时间戳格式**：见 [03](03-数据库映射与陷阱.md)，硬切换或给 Flask 上 ISO 兼容类型，二选一。
2. **会话不互通**：切换需用户重新登录一次（密码不用改）。
3. **内存限频**：单进程有效；多实例部署要换 Redis。
4. **SQLite 并发写**：高并发写入是 SQLite 固有短板；长期建议迁 Postgres（[03 §7](03-数据库映射与陷阱.md)）。
5. **账户微服务 fail-closed**：迁移时务必保留，否则远端故障会静默扣鱼/发鱼不一致。
