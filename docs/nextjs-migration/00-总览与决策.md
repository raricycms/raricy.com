## 1. 关键架构决策（我替你做的判断）

1. **框架**：Next.js 15（App Router）+ React 19 + TypeScript。理由：SSR/SEO 对博客站是刚需，
   App Router 的 Server Components 能把“Jinja 服务端渲染”几乎 1:1 平移。

2. **ORM**：Prisma，`provider = sqlite`，**直连现有同一个 SQLite 库**。
   → **零数据迁移**：不导数据、不换库，Next 读的就是 Flask 写的那张 `db.db`。
   这是最安全的切换路径，也是你特别叮嘱“注意数据库调用方式”的核心答复。

3. **数据库 schema 归属不变**：仍由 **Flask-Migrate/Alembic** 管理（`alembic_version` 表保留）。
   **绝不**对该库跑 `prisma migrate` —— Prisma 仅作读写客户端。见 [03](03-数据库映射与陷阱.md)。

4. **认证**：密码哈希与 werkzeug **完全兼容**（scrypt / pbkdf2 都支持），用户**无需重置密码**；
   会话换成 JWT（itsdangerous 的 Flask session 无法与 JWT 互通，代价是切换时**需重新登录一次**）。
   见 [04](04-认证与会话迁移.md)。

5. **过渡期共存**：`next.config.mjs` 用 `rewrites` 把 `/auth/avatar/*`、`/image/*` 回源 Flask，
   头像与图床不必第一天就迁。允许 Flask 与 Next 并行灰度。

6. **限频**：Flask 里点赞/评论/投票/图床各写了一份内存限频，这里**抽成一个共享工具**
   `src/lib/rate-limit.ts`（正是 CLAUDE.md 的建议）。多实例部署时应换 Redis。
