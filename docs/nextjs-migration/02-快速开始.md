# 快速开始：把 web-next 跑起来

## 前置

- Node ≥ 20（本机实测 22.23）
- 现有 `instance/database/db.db` 存在

## 步骤

```bash
cd web-next

# 1. 安装依赖（已装则跳过）
npm install

# 2. 生成规整后的开发数据库副本（把线上 SQLite 复制成 prisma/dev.db 并转 ISO 时间戳）
npm run db:normalize

# 3. 配置环境变量
cp .env.example .env
#   .env 里：
#   DATABASE_URL="file:./dev.db"              ← 开发用副本；上线改指线上库
#   SECRET_KEY="<与 Flask .env 相同>"          ← 会话签名 + 加密字段一致性
#   FLASK_ORIGIN="http://127.0.0.1:5050"       ← 头像/图床过渡期回源

# 4. 生成 Prisma Client
npm run prisma:generate

# 5. 开发模式
npm run dev            # http://localhost:3000

# 或生产构建 + 启动
npm run build && npm start
```

## 冒烟测试（都应通过）

```bash
# 列表（SSR 读真实数据）
curl -s http://127.0.0.1:3000/api/blogs | head -c 300

# 未登录
curl -s http://127.0.0.1:3000/api/auth/me            # → {"user":null}

# 登录（用真实用户；开发副本里可先设个已知密码）
curl -s -c /tmp/ck.txt -X POST http://127.0.0.1:3000/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"<你的用户名>","password":"<密码>"}'

# 带 cookie 取当前用户
curl -s -b /tmp/ck.txt http://127.0.0.1:3000/api/auth/me

# 点赞（需登录）
curl -s -b /tmp/ck.txt -X POST http://127.0.0.1:3000/api/blogs/<博客ID>/like
```

## 在开发副本上设一个已知密码（仅用于本地验证）

```bash
# 用真实 werkzeug 生成哈希（证明跨栈互通），写进 dev.db 副本
HASH=$(../venv/bin/python -c "from werkzeug.security import generate_password_hash as g; print(g('test123'))")
sqlite3 prisma/dev.db "UPDATE users SET password_hash='$HASH' WHERE username='<用户名>';"
```

## 常用脚本

| 命令 | 作用 |
|---|---|
| `npm run dev` | 开发服务器（热重载）|
| `npm run build` / `npm start` | 生产构建 / 启动 |
| `npm run prisma:generate` | 生成 Prisma Client |
| `npm run prisma:studio` | 可视化浏览数据库 |
| `npm run db:normalize` | 生成/刷新规整后的 dev.db |
