### Phase 1.5：写路径 fail-closed 化（✅ 完成 2026-07-04）

**目标**：避免"本地扣款但远端没记账"的不一致窗口。所有写操作改为：本地先收集变更 → 远端同步成功后才 commit 本地 → 远端失败则 rollback 整个本地事务。

**完成情况：**

| 写路径 | 状态 | API 错误码 / CLI 退出码 |
|--------|------|------------------------|
| 投喂（`feed_fish_service.feed_fish`） | ✅ fail-closed（2026-07-04 完成） | HTTP 503 |
| 签到（`checkin.claim_fortune`） | ✅ fail-closed（2026-07-04 完成） | HTTP 503 |
| 管理员 CLI grant（`flask fish grant`） | ✅ fail-closed（2026-07-04 完成） | exit 2 |
| 管理员 CLI deduct（`flask fish deduct`） | ✅ fail-closed（2026-07-04 完成） | exit 2 |

**统一模式**：

```python
# 所有写路径的标准模板（以签到为例）
add_fish(user_id, amount, 'checkin', ..., auto_commit=False)

try:
    current_app.account_client.transfer(...)
except AccountClientError as e:
    db.session.rollback()      # 撤销本地变更
    logger.warning(...)
    raise                      # API 路由 → 503；CLI → exit 2
db.session.commit()             # 远端成功才 commit 本地
```

**验收标准（全部达成）：**
- ✅ 所有写路径：远端不可用时本地零变更，HTTP 503 / exit 2 明确告知
- ✅ 不再出现"博客鱼干数增加但用户鱼干未扣"的状态
- ✅ 存量差异已修复（2 个用户），全部 6 个用户 0 差异

**已知代价**：
- 远端账户服务挂了，所有鱼干相关操作（投喂、签到、管理员发放）全部瘫痪。
- 需要在账户服务前加 LB + 健康检查，把单点故障降到最低。

   ```python
   def use_remote_fish(user_id: str) -> bool:

       if not current_app.config.get("FISH_REMOTE_ENABLED", False):
   ```

## 2. 未来扩展：模拟炒股

### 2.1 概述

模拟炒股是用户用小鱼干作为交易货币，在模拟市场中进行股票买卖。不涉及真实资金，但交易逻辑要尽量仿真。

### 2.2 新增数据模型

```sql
-- 股票持仓表
CREATE TABLE stock_positions (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    account_id  UUID NOT NULL REFERENCES accounts(id),
    symbol      VARCHAR(20) NOT NULL,     -- 股票代码，如 "600036.SH"
    quantity    BIGINT NOT NULL,          -- 持仓数量（整型，1 股 = 1000 单位）
    avg_cost    BIGINT NOT NULL,          -- 平均成本（内部金额单位）
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),

    CONSTRAINT uq_account_symbol UNIQUE (account_id, symbol)
);

-- 订单表
CREATE TABLE orders (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    account_id  UUID NOT NULL REFERENCES accounts(id),
    symbol      VARCHAR(20) NOT NULL,
    side        VARCHAR(4) NOT NULL CHECK (side IN ('BUY', 'SELL')),
    order_type  VARCHAR(10) NOT NULL DEFAULT 'LIMIT' CHECK (order_type IN ('MARKET', 'LIMIT')),
    price       BIGINT,                   -- 限价（内部金额单位），市价单为 NULL
    quantity    BIGINT NOT NULL,          -- 委托数量
    filled_qty  BIGINT NOT NULL DEFAULT 0, -- 已成交数量
    status      VARCHAR(10) NOT NULL DEFAULT 'OPEN'
                CHECK (status IN ('OPEN', 'PARTIAL', 'FILLED', 'CANCELLED')),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_orders_account ON orders(account_id, status);
CREATE INDEX idx_orders_symbol ON orders(symbol, status);

-- 成交记录表
CREATE TABLE trades (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    buy_order_id UUID NOT NULL REFERENCES orders(id),
    sell_order_id UUID NOT NULL REFERENCES orders(id),
    symbol      VARCHAR(20) NOT NULL,
    price       BIGINT NOT NULL,          -- 成交价
    quantity    BIGINT NOT NULL,          -- 成交数量
    buyer_account_id UUID NOT NULL REFERENCES accounts(id),
    seller_account_id UUID NOT NULL REFERENCES accounts(id),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### 2.3 撮合模式

模拟炒股不需要实时撮合。推荐两种模式：

**模式 A：定时批量撮合（推荐）**

```
每 1 分钟（或 5 分钟）：
1. 收集所有 OPEN 订单
2. 从外部 API 获取最新行情价
3. 市价单 → 直接按行情价成交
4. 限价单 → 行情价 <= 买入限价 或 >= 卖出限价 则成交
5. 生成 trades + 更新 positions + 调用 transfer() 划转小鱼干
```

用 Celery Beat 调度：

```python
@app.task

    # 3. 匹配成交
    # 4. 写 trades、更新 positions、更新订单状态
    # 5. 调用 transfer_service.transfer() 划转小鱼干
```

**模式 B：即时撮合（简单版）**

用户下单时立即按当前行情价成交。不需要订单簿，适合初期用户少的时候。

### 2.4 行情源

国内免费行情 API：

| 来源 | 接口 | 限制 |
|------|------|------|
| 新浪财经 | `hq.sinajs.cn` | 免费、稳定、覆盖 A 股 |
| 腾讯财经 | `qt.gtimg.cn` | 免费、实时 |
| EastMoney | `push2.eastmoney.com` | 免费、数据全 |

示例：

```python
import httpx

async def get_stock_price(symbol: str) -> float:
    """获取 A 股实时价格"""
    # 新浪接口格式
    url = f"http://hq.sinajs.cn/list={symbol}"
    async with httpx.AsyncClient() as client:
        resp = await client.get(url)
        # 解析返回文本，提取当前价
        # var hq_str_sh600036="招商银行,42.50,..."
        price = float(resp.text.split('"')[1].split(',')[3])
        return price
```

### 2.5 与小魚干的关系

- **交易货币**：买卖股票用小鱼干结算
- **手续费**：每笔成交扣 0.1% 小鱼干，归系统账户（类似投喂消耗）
- **盈亏**：`(卖出价 - 买入价) × 股数 - 手续费`

```python
# 撮合成交后
# 买方：扣减小鱼干（buy_price × qty + fee）→ 系统账户
# 卖方：系统账户 → 增加小鱼干（sell_price × qty - fee）
# 手续费：买方 + 卖方的手续费 → 系统账户

transfer_service.transfer(
    from_user_id=buyer_id,
    to_user_id="raricy-blog-system",
    amount=buy_price * qty + fee,
    entry_type="stock_buy",
    description=f"买入 {symbol} {qty} 股 @ {buy_price}",
)

transfer_service.transfer(
    from_user_id="raricy-blog-system",
    to_user_id=seller_id,
    amount=sell_price * qty - fee,
    entry_type="stock_sell",
    description=f"卖出 {symbol} {qty} 股 @ {sell_price}",
)
```

### 3.3 交易所式挂单撮合

如果用户量大了，可以从定时批量撮合升级为：

```
订单簿（Order Book）:
  BUY  [price: 42.50, qty: 100] → [price: 42.60, qty: 200]
  SELL [price: 42.70, qty: 150] → [price: 42.80, qty: 50]

用户下单 → 检查对手盘 → 有匹配就立即成交 → 剩余挂单入簿
```

这需要 Redis Sorted Set 做订单簿（按价格排序），撮合引擎用 Lua 脚本在 Redis 中原子执行。但这是很远的事情了。
