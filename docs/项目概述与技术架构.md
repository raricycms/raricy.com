# 项目概述与技术架构

## 项目简介

聪明山（raricy.com）是一个基于 Flask 的个人网站平台，集成了博客、故事展示、工具集、剪贴板、游戏等多个功能模块。

**技术栈：** Python / Flask + SQLAlchemy + Jinja2 + SCSS + 原生 JavaScript

## 应用架构

### 应用工厂

`run.py` → `app/__init__.py:create_app()` 创建 Flask 实例，按顺序：

1. 加载配置
2. 注册 Jinja2 过滤器/全局函数
3. 注册蓝图
4. 初始化扩展
5. 注册 CLI 命令

### 蓝图（Blueprint）

所有蓝图在 `app/web/__init__.py:register_blueprints()` 中统一注册：

| 蓝图 | URL 前缀 | 用途 |
|------|----------|------|
| `home_bp` | `/` | 首页、robots.txt、联系页面、邀请码生成 |
| `auth_bp` | `/auth` | 登录、注册、认证、资料、用户管理 |
| `blog_bp` | `/blog` | 博客 CRUD、管理后台、API（点赞/评论）、爬虫 API |
| `notifications_bp` | `/notifications` | 通知中心 |
| `tool_bp` | `/tool` | 工具页面 |
| `story_bp` | `/story` | 故事模块 |
| `game_bp` | `/game` | 游戏模块 |
| `clip_bp` | `/clipboard` | 剪贴板模块 |
| `sitemap_bp` | `/sitemap` | 站点地图 |
| `audit_bp` | `/audit` | 管理员操作日志公示 |
| `error_bp` | `/error` | 错误页面 |
| `test_bp` | （仅 DEBUG 模式） | 测试路由 |

### 扩展

`app/extensions/__init__.py` 初始化：

- **SQLAlchemy (db)** — ORM，使用 Flask-Migrate 管理迁移
- **Flask-Login** — 会话管理，配合 `session_version` 机制实现会话失效
- **Flask-Migrate** — 数据库迁移（不使用 `db.create_all()`）
- **Turnstile** — Cloudflare 人机验证
- **ProxyFix** — 信任 Nginx 反向代理头，确保 `url_for` 生成正确 HTTPS URL

### 配置

`app/config.py` — 通过 `CONFIG_TYPE` 环境变量切换 `DevelopmentConfig` / `ProductionConfig` / `TestingConfig`。所有敏感配置从 `.env` 读取。

## 数据模型

所有业务模型使用 UUID 字符串作主键（`user.id`、`blog.id`、`comment.id`）。`Category.id` 和审计日志使用自增整数。

### 核心模型

**User** (`app/models/user.py`) — 角色体系：`user` → `core` → `admin` → `owner`：
- `is_core_user` — 已通过邀请码认证
- `has_admin_rights` — 拥有管理员权限
- `is_owner` — 站长
- 禁言系统：`is_banned` / `ban_until` / `ban_reason`
- 通知偏好设置
- 会话版本机制（`session_version`）：修改密码或被踢下线时递增，`user_loader` 检测不匹配则清除会话

**InviteCode** — 邀请码，注册时提供则直接升级为 core 用户。

**UserBan** — 禁言历史记录。

**Blog + BlogContent** (`app/models/blog.py`) — 一对一分表设计：
- `Blog` — 元信息（标题、摘要、作者、分类、精选、点赞数）
- `BlogContent` — Markdown 正文
- 分表避免正文影响列表查询性能

**BlogLike** — 点赞记录，唯一约束 `(blog_id, user_id)`。

**Category** (`app/models/category.py`) — 二级分类层级（`parent_id` 自关联）：
- 排序、启用/禁用
- 可从"全部文章"排除
- 可设为管理员专属发文

**BlogComment + CommentLike** (`app/models/comment.py`) — 楼中楼评论：
- `parent_id` / `root_id` 构建树形结构
- 审核状态：`pending` / `approved` / `hidden`
- 软删除（`is_deleted`）

**Notification** (`app/models/notification.py`) — 用户通知，支持已读/未读、批量操作。

**ClipBoard + ClipText** (`app/models/clipboard.py`) — 剪贴板，短 ID（8 位 base62），支持公开/私有。

**AdminActionLog + AdminActionAppeal** (`app/models/audit.py`) — 管理员操作日志和申诉系统。

## 分层设计（以博客模块为例）

`app/web/blog/` 采用分层架构：

| 层 | 文件/目录 | 职责 |
|----|-----------|------|
| 视图 | `views.py` | 用户端路由（列表、详情、上传、编辑、删除） |
| API | `api_views.py` | 前端 API（点赞切换、评论 CRUD、点赞者列表） |
| 管理 | `admin_views.py` | 管理员路由（仪表盘、栏目管理、文章管理） |
| 爬虫 | `spider_api.py` | 爬虫 API（允许搜索引擎爬取内容） |
| 服务 | `services/` | 业务逻辑层（BlogService、CommentService、LikeService、CategoryService） |
| 验证 | `validators/` | 输入验证层 |
| 工具 | `utils/` | 响应格式化、禁言检查等 |

## 通知系统

`app/service/notifications.py`：
- 通知发送、查询、已读管理、管理员群发
- 发送前检查用户通知偏好（除非 `force=True`）
- 支持的通知类型：文章被点赞/评论、评论被回复/点赞、管理员操作通知等

## 认证与权限

`app/extensions/decorators.py`：

| 装饰器 | 要求 |
|--------|------|
| `authenticated_required` | 登录 + 已通过邀请码认证（core 及以上角色） |
| `admin_required` | 管理员权限 |
| `owner_required` | 站长权限 |

## 前端架构

- **模板引擎：** Jinja2，基础模板 `base.html`，其余模板按蓝图分目录
- **样式：** SCSS 源码在 `app/static/scss/`，按 `abstracts/base/layout/components/utilities/pages` 分层
- **JavaScript：** 原生 JS，位于 `app/static/js/`
- **主题：** 支持亮色/暗色主题切换，CSS 自定义属性，本地存储偏好
- **缓存破坏：** 全局函数 `static_url(filename)` 在静态资源 URL 后追加文件修改时间

## 关键约定

- 博客删除为软删除（`ignore=True`），不物理删除文件
- 所有用户输入通过验证层（Validators）处理
- API 响应统一格式：`{ code: 200, message: "...", ...data }`
- 不使用 `db.create_all()`，所有表变更通过 Flask-Migrate 迁移
- 日志中使用 ANSI 颜色码输出彩色终端提示
