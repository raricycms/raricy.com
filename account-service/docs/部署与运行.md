# 部署与运行

## 1. 环境要求

| 组件 | 版本要求 |
|------|---------|
| Python | 3.12+ |
| pip / uv | 最新稳定版 |

**无需外部数据库。** account-service 使用 SQLite（`account.db`），一个文件搞定，与博客项目的 SQLite 部署方式一致。

## 2. 快速初始化

```bash
cd account-service

# 创建虚拟环境
python -m venv .venv
source .venv/bin/activate      # Linux / macOS
# .venv\Scripts\activate       # Windows

# 安装依赖
pip install -e .

# 配置环境变量
cp .env.example .env
# 编辑 .env，填入 INTERNAL_TOKEN（DATABASE_URL 已有默认值，无需改动）

# 运行迁移
alembic upgrade head

# 创建系统账户（获取 API Key）
python scripts/seed.py
```

## 3. 配置文件（.env）

```env
# 数据库（SQLite — 默认即可，无需额外配置）
DATABASE_URL=sqlite+aiosqlite:///account.db

# 服务
DEBUG=true                      # 开发环境设为 true（错误响应包含异常类型名）
SERVICE_NAME=account-service

# 内部认证 — 与博客 .env 中的值必须一致
INTERNAL_TOKEN=<替换为强随机字符串>

# 如需切换 PostgreSQL：
#   1) pip install asyncpg
#   2) 修改 DATABASE_URL 为 postgresql+asyncpg://user:pass@host:5432/account
#   3) 重启服务

# 系统账户（可选，有默认值）
# SYSTEM_ACCOUNT_ID=00000000-0000-0000-0000-000000000000
# SYSTEM_USER_ID=raricy-blog-system

# API Key（可选，有默认值）
# API_KEY_PREFIX=fish_sk_

# 幂等（可选，有默认值）
# IDEMPOTENCY_EXPIRY_HOURS=24
```

### 生成 INTERNAL_TOKEN

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

将此值填入 account-service 的 `.env` 和博客的 `.env`（`ACCOUNT_SERVICE_INTERNAL_TOKEN`），两边的值必须一致。

## 4. 运行

### 开发模式

```bash
uvicorn app.main:app --reload --port 8000
```

API 文档自动生成：
- Swagger UI：http://localhost:8000/docs
- ReDoc：http://localhost:8000/redoc

### 生产模式

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

> **注意：** 不要使用 `--workers` 多进程。SQLite 不支持并发写入，多 worker 进程同时访问同一个 `account.db` 文件会导致 "database is locked" 错误。如需更高并发，考虑将数据库迁移至 PostgreSQL 后再启用多 worker。

## 5. Nginx 反向代理

```nginx
server {
    listen 80;
    server_name account.internal;   # 内网域名，不暴露公网

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

**重要：** account-service 不应直接暴露在公网。所有请求应来自博客后端，经过内网或本地回环。

## 6. systemd 服务

`/etc/systemd/system/account-service.service`：

```ini
[Unit]
Description=raricy.com Account Service
After=network.target

[Service]
Type=simple
User=raricy
Group=raricy
WorkingDirectory=/opt/raricy/account-service
EnvironmentFile=/opt/raricy/account-service/.env
ExecStart=/opt/raricy/account-service/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now account-service
```

## 7. 数据库维护

### 创建迁移

```bash
# 修改模型后
alembic revision --autogenerate -m "描述你的变更"

# 检查生成的迁移文件
# 应用迁移
alembic upgrade head
```

### 幂等键清理

幂等键 24 小时后自动过期，但不会自动从数据库中物理删除。可设置 cron 定期清理：

```sql
DELETE FROM idempotency_keys WHERE expires_at < NOW();
```

建议每天凌晨执行一次。

### 备份

```bash
# SQLite 备份就是复制文件
cp account.db account_backup_$(date +%Y%m%d).db
```

## 8. 健康检查

```bash
curl http://localhost:8000/health
```

响应示例：

```json
{
  "status": "ok",
  "version": "0.1.0",
  "db": "connected"
}
```

- `status: "ok"` — 服务正常，数据库可连接
- `status: "degraded"` — 服务运行中但数据库不可达

## 9. 日志与调试

`DEBUG=true` 时，500 错误响应包含异常类型名：

```json
{
  "code": 500,
  "message": "Internal server error",
  "detail": {"type": "IntegrityError"},
  "request_id": "a1b2c3d4-..."
}
```

生产环境应设 `DEBUG=false`，此时 `detail` 为 `null`。通过 `request_id` 在应用日志中关联排查。

uvicorn 访问日志已包含请求方法、路径、状态码和耗时，可在反向代理层进一步采集。
